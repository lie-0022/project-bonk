## Elevator Pitch

> 귀엽고 밝은 중세판타지 세계관에서 검사·마법사·거너 중 하나를 골라 몰려오는 몬스터를 쓸어버리는 3D 뱀파이어 서바이벌라이크.
> 게임 중 획득한 무기로 빌드를 완성하고, 3개 스테이지를 클리어하며 점점 강해지는 파워 트립을 경험하세요.
> MegaBonk에서 영감을 받아 학교 프로젝트로 제작합니다.

## Core Fantasy

귀여운 영웅이 수백 마리의 몬스터를 상대로 점점 압도적으로 강해지는 파워 트립.
시작은 소박한 검사지만, 매 레벨업마다 내가 직접 선택한 무기와 스펙이 쌓여 나만의 빌드가 완성된다.
3개 스테이지를 넘어갈수록 몬스터는 강해지지만, 나는 더 빠르게 강해진다.

---

## Unique Hook

MegaBonk의 3D 뱀서라이크 구조를 학습하고, **귀엽고 밝은 중세판타지 비주얼**로 재해석한다.
자동공격(가장 가까운 적) + 근접공격(바라보는 방향)의 이중 전투 구조로 조작감을 살린다.
스테이지 클리어마다 스펙이 초기화되는 구조로 매 스테이지가 새로운 도전이 된다.

## Player Experience Analysis (MDA Framework)

### Target Aesthetics (What the player FEELS)

| Aesthetic | Priority | How We Deliver It |
| ---- | ---- | ---- |
| **Sensation** (sensory pleasure) | 2 | 타격 이펙트, 등급 연출, 피격 피드백 (진동·피·데미지 표시) |
| **Fantasy** (make-believe, role-playing) | 3 | 귀여운 영웅이 군중을 쓸어버리는 판타지 |
| **Narrative** (drama, story arc) | 5 | 스테이지 진입·클리어 컷신, 간단한 스크립트 스토리 |
| **Challenge** (obstacle course, mastery) | 4 | 스테이지별 난이도 상승, 보스 처치 |
| **Fellowship** (social connection) | N/A | 싱글플레이 |
| **Discovery** (exploration, secrets) | 5 | 등급별 아이템 조합, 제단 효과 발견 |
| **Expression** (self-expression, creativity) | 1 | 캐릭터 선택 + 매 런마다 다른 빌드 |
| **Submission** (relaxation, comfort zone) | N/A | 긴장감 기반 게임 |

### Key Dynamics (Emergent player behaviors)

- 플레이어는 "이번엔 어떤 캐릭터로, 어떤 빌드로 가볼까"를 생각하며 런을 반복한다
- 스테이지 클리어 후 스펙 초기화가 다음 스테이지의 빌드 선택을 더 신중하게 만든다
- 등급이 올라가는 연출(상자 오픈 등)이 도파민을 자극해 재플레이를 유도한다

### Core Mechanics (Systems we build)

1. **플레이어 이동 + 대시 + 점프** — WASD 이동, Shift 대시(무적), Space 점프, 낙하 데미지
2. **오빗 카메라** — 마우스로 카메라만 회전, 캐릭터는 이동방향을 바라봄
3. **자동 공격** — 캐릭터별 기본 공격 자동 발동 (검사: 근접 범위 / 마법사·거너: 투사체)
4. **무기 획득 시스템** — 게임 중 무기 드롭·상자에서 새 무기 획득, 장착해 공격 방식 추가·강화
5. **웨이브 스폰** — 시간 기반으로 적 수·강도 증가, 스테이지마다 난이도 상승
6. **적 AI** — 플레이어 추적, 원거리 공격 예고(빨간 범위), 벽 타고 올라옴
7. **골드 시스템** — 인게임 골드로 상자 오픈·상인 구매
8. **스테이지 진행** — 3스테이지(3분/5분/7분), 클리어 시 스펙 초기화 후 다음 스테이지

## Characters

캐릭터 선택은 메인 메뉴에서 이루어진다. 각 캐릭터는 기본 공격 방식이 다르며, 게임 중 획득하는 무기도 캐릭터 특성에 맞게 구성된다.

| 캐릭터 | 기본 공격 방식 | 특성 | 구현 우선순위 |
|--------|--------------|------|-------------|
| **마법사** | 투사체 (마법 볼트) | 원거리, 범위 마법 | MVP (1순위) |
| **검사** | 근접 범위 공격 (전방 부채꼴) | 근접 강함, 생존력 높음 | MVP (2순위) |
| **거너** | 투사체 (총알) | 원거리 빠른 연사 | Vertical Slice |

> 디테일한 캐릭터별 수치·무기 구성은 별도 캐릭터 GDD에서 정의 (TBD)

## Stage Structure

| 스테이지 | 제한시간 | 난이도 | 특징 |
|----------|---------|--------|------|
| Stage 1 | 3분 | 쉬움 | 근접 몬스터 위주, 튜토리얼 역할 |
| Stage 2 | 5분 | 보통 | 원거리 공격 몬스터 등장 |
| Stage 3 | 7분 | 어려움 | 엘리트·보스 몬스터, 최종 도전 |

- 스테이지 클리어 시 모든 스펙 초기화
- 클리어 조건: **보스 처치** (확정). 보스는 해당 스테이지 일정 시간 경과 시 자동 소환
- 보스 처치 → 다음 스테이지 이동 제단 등장

## Core Loop

### Moment-to-Moment (30초)
몬스터들이 몰려오는 가운데 WASD + 마우스로 이동·회피하며 자동공격이 가장 가까운 적을 처치한다.
대시로 위험 회피, 점프로 지형 활용. 몬스터 처치 시 경험치·골드 획득.

### Short-Term (3-7분 / 스테이지 1회)
몬스터 처치 → 무기 드롭·상자에서 새 무기 획득 → 빌드 강화 → 보스 처치 → 스테이지 클리어.
상인으로 추가 구매 가능.

### Session-Level (15-18분 / 3스테이지)
Stage 1(쉬움) → Stage 2(보통) → Stage 3(어려움) → 최종 클리어 또는 사망 → 결과 화면(킬 수·생존 시간·랭킹).

### Long-Term Progression
런 완료 시 외부 재화 획득 → 새 캐릭터·아이템 해금.

## Game Pillars

### Pillar 1: 시각적 매력
SD/MD 캐릭터와 밝은 중세판타지 세계관이 첫인상을 결정한다. 아이콘·폰트·등급 연출 모두 컨셉에 맞게.

*Design test*: 스크린샷만 봐도 "귀엽고 밝은 판타지 게임"임을 알 수 있어야 한다.

### Pillar 2: 빌드 정체성
매 런마다 선택한 무기·스펙 조합이 눈에 띄게 다른 플레이 스타일을 만들어야 한다.

*Design test*: 모든 빌드가 "그냥 강하게"만 느껴진다면 → 무기·스펙 간 차별화를 강화한다.

### Pillar 3: 파워 에스컬레이션
시작은 소박하게, 스테이지 후반은 압도적으로. 스펙 초기화 후에도 이전보다 빨리 강해지는 숙련감.

*Design test*: Stage 3는 Stage 1보다 확연히 강하고 화려해야 한다.

### Anti-Pillars (What This Game Is NOT)

- **NOT 복잡한 스토리**: 컷신은 짧고 간결하게. 스토리보다 게임플레이가 우선
- **NOT 멀티플레이**: 싱글플레이에 집중
- **NOT 완벽한 그래픽**: 에셋 스토어 활용, 폴리싱은 로직 완성 후

## Risks and Open Questions

### Design Risks
- 스테이지 초기화가 너무 가혹하게 느껴질 수 있음 → 스테이지 사이 일부 아이템 유지 옵션 검토
- 3캐릭터 밸런싱이 복잡할 수 있음 → MVP는 검사 1캐릭터만

### Technical Risks
- 오빗 카메라 + 이동 방향 회전 구현 복잡도 → 최우선 구현 항목
- 다수 적(50+) 동시 처리 성능 → Object Pooling 초기부터 적용

### Scope Risks
- 10주 중 로직 1-2주, 아트·UI 병행 진행 → 기능 완성 우선, 아트는 에셋 스토어 활용
- 3캐릭터 모두 구현 욕심 → MVP는 검사만, 나머지는 Vertical Slice

### Open Questions
- Stage 클리어 조건: **보스 처치로 확정** (보스는 스테이지 일정 시간 경과 시 자동 소환)
- 랭킹 기준: 생존시간, 킬 수, 클리어 속도 복합 점수?

## MVP Definition

**Core hypothesis**: "오빗 카메라 + 자동공격 + 레벨업 선택이 15분짜리 재미있는 런을 만드는가?"

**MVP 구현 목록**:
1. 오빗 카메라 (마우스 회전)
2. 플레이어 이동 + 대시(Shift) + 점프(Space)
3. 마법사 기본 공격 (투사체 자동 발사) — 검사 (근접 범위) 2순위
4. HP 시스템 + 피격 피드백
5. 몬스터 AI + 웨이브 스포너
6. 게임 상태 관리 (시작/플레이/게임오버)
7. XP + 레벨업
8. 무기 획득 시스템 기본 (드롭·장착)
9. 골드 + 상자 기본 구현
10. Stage 1 완성 (5웨이브, 기본 몬스터)
11. 결과 화면 (킬 수, 생존 시간)

**Explicitly NOT in MVP**:
- 마법사·거너 캐릭터
- 캐릭터 선택 화면 (메인 메뉴)
- 상인 시스템
- 보스 몬스터
- 미니맵
- Stage 2·3
- 컷신·스토리
- 랭킹 시스템
